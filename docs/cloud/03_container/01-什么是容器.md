# 容器技术介绍

## 容器的发展路程
容器的本质就是一种轻量级的虚拟化方案
::: details 单机时代
一个计算机硬件上只能跑一个系统，这个时候面临的问题是如何提升单机的资源利用率。
解决方案：同时在一个系统上跑多个应用程序（mysql，tomcat，nginx 都跑到同一台机器）
新问题：应用程序之间的资源竞争，单节点故障问题
:::

::: details 集群时代
一个计算机仍然只能运行一个系统，为了解决单机时代出现的新问题（资源竞争，单点故障），将多个应用程序分开部署。
解决方案：一个机器只部署一个应用，物理隔离，解决了资源竞争，单点故障的问题。
新问题：回归到原点，资源利用率依旧不高，花了很多钱，但是有很多闲置资源。
:::

::: details 虚拟化时代
一个计算机可以同时在宿主机上运行多个虚拟系统（VMware，KVM，Qemu ...），在每一个虚拟机上部署应用程序。解决了资源竞争，和资源利用率不足的问题。
可以在多个物理机器上分割出若干虚拟系统，然后限制硬件资源。之后分别部署多套应用程序副本，这样也解决了单点故障的问题。
新问题：过于臃肿，启动慢，占用资源大
:::

::: details 容器时代
容器时代核心要做的就是解决虚拟化时代的臃肿，启动慢，占用资源大的问题。
容器如何做到比虚拟机轻？轻量级镜像，共享宿主机内核
容器如何限制资源？cgroup
容器如何资源隔离？（网络，主机命名空间，进程，挂载点，用户和组，消息通信）
:::
